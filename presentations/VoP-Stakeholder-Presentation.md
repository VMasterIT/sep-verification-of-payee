## Слайд 2: Проблема

### Що не так з поточною ситуацією?

**Проблеми:**

😞 **Шахрайство**
- Phishing атаки

😞 **Помилки платників**

😞 **Довіра до платежів**

- Відмова від онлайн-переказів
- Перехід на готівку

## Слайд 3: Рішення — VoP

### Що таке Verification of Payee?

**VoP — це система, яка перевіряє, чи збігається ім'я отримувача з власником рахунку ПЕРЕД відправкою платежу**

**Як це працює:**

1. Платник вводить IBAN та ім'я отримувача
2. Банк відправляє запит до VoP системи
3. VoP перевіряє, чи збігається ім'я з власником рахунку
4. Платник бачить результат:
   - ✅ **MATCH** — Ім'я співпадає, можна платити
   - ⚠️ **CLOSE_MATCH** — Часткове співпадіння, перевірте ім'я
   - ❌ **NO_MATCH** — Ім'я не співпадає, НЕ платіть!

## Слайд 4: Приклад використання

### Сценарій: Переказ зарплати

**Без VoP:**
```
Співробітниця: Марія ІВАНЕНКО
HR вводить:    Марія ІВАНЧУК (помилка!)
IBAN:          UA21...001 (рахунок ІВАНЕНКО)

❌ Гроші йдуть на рахунок ІВАНЕНКО,
   але в платежі вказано ІВАНЧУК
→ Плутанина, скарги, повернення
```

**З VoP:**
```
Співробітниця: Марія ІВАНЕНКО
HR вводить:    Марія ІВАНЧУК (помилка!)
IBAN:          UA21...001 (рахунок ІВАНЕНКО)

⚠️ VoP: "NO_MATCH - Власник рахунку: ІВАНЕНКО Марія Петрівна"
→ HR виправляє ім'я ПЕРЕД відправкою
→ Проблеми немає!
```

## Слайд 6: Переваги VoP

### Для банків

✅ **Зниження шахрайства** → менше збитків
✅ **Менше скарг** → менше навантаження на support
✅ **Комплаєнс з EU** → готовність до інтеграції з ЄС
✅ **Конкурентна перевага** → перші в Україні

### Для клієнтів

✅ **Захист від шахрайства** → спокій та безпека
✅ **Впевненість у платежах** → немає страху помилитись
✅ **Швидке виявлення помилок** → до відправки грошей
✅ **Зручність** → автоматична перевірка

### Для НБУ

✅ **Безпека платіжної системи** → менше інцидентів
✅ **Довіра до СЕП** → більше електронних платежів
✅ **Міжнародні стандарти** → інтеграція з EU VoP

## Слайд 7: Архітектура VoP

### Як влаштована система

```
┌─────────────┐                    ┌─────────────┐
│   Банк A    │                    │   Банк B    │
│ (Requester) │                    │ (Responder) │
│             │                    │             │
│ Mobile App  │                    │ CBS/Database│
└──────┬──────┘                    └──────▲──────┘
       │                                  │
       │ 1. VoP Request                  │ 3. Check
       │ (IBAN + Name)                   │    name
       │                                  │
       v                                  │
┌─────────────────────────────────────────┴──┐
│          VoP Router (НБУ)                  │
│  ┌──────────────────────────────────────┐  │
│  │ Directory Service (IBAN → Bank)     │  │
│  └──────────────────────────────────────┘  │
│  ┌──────────────────────────────────────┐  │
│  │ Routing + Authentication + Logging   │  │
│  └──────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
       │
       │ 2. Forward request
       └────────────────────────────────────►
```

**Ключові компоненти:**
- **VoP Router (НБУ)** — центральна маршрутизація
- **VoP Requester** — банк-ініціатор платежу
- **VoP Responder** — банк-отримувач платежу

## Слайд 8: Name Matching

### Як перевіряється ім'я?

**Алгоритми:**

1. **Levenshtein Distance** — вимірює кількість змін (вставки, видалення, заміни)
2. **Jaro-Winkler Distance** — оптимізований для коротких рядків, враховує префікс

**Приклади:**

| Введено | В системі | Score | Результат |
|---------|-----------|-------|-----------|
| ШЕВЧЕНКО ТАРАС | ШЕВЧЕНКО ТАРАС | 100% | ✅ MATCH |
| ШЕВЧЕНКО ТАРАС | ШЕВЧЕНКО ТАРАК | 93% | ⚠️ CLOSE_MATCH |
| ШЕВЧЕНКО Т.Г. | ШЕВЧЕНКО ТАРАС ГРИГОРОВИЧ | 100% | ✅ MATCH (initials) |
| ШЕВЧЕНКО ТАРАС | ІВАНЕНКО ПЕТРО | 20% | ❌ NO_MATCH |

**Пороги:**
- ≥95% → MATCH
- 75-94% → CLOSE_MATCH
- <75% → NO_MATCH

## Слайд 9: Безпека

### Як захищена система?

🔐 **mTLS (Mutual TLS)**
- Взаємна автентифікація банків
- Сертифікати від АЦСЬК
- TLS 1.2+ з сильними шифрами

🔐 **OAuth 2.0 + FAPI**
- Токени доступу з коротким терміном дії
- Financial-grade API стандарт
- Scope-based authorization

🔐 **Audit Logging**
- Всі запити логуються
- GDPR compliance (retention 90 днів)
- Моніторинг та alerting 24/7

🔐 **Data Minimization**
- Зберігається мінімум даних
- Імена хешуються в логах
- Немає централізованої БД клієнтів
