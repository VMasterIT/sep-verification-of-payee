## 1. ТЕРМІНИ ТА ВИЗНАЧЕННЯ

**1.1. VoP (Verification of Payee)** – система перевірки реквізитів отримувача платежу перед відправленням платіжного повідомлення до СЕП НБУ.

**1.2. VoP Router** – централізована система маршрутизації VoP запитів, яка забезпечує зв'язок між банками-учасниками.

**1.3. VoP Request** – запит на перевірку реквізитів отримувача платежу.

**1.4. VoP Response** – відповідь з результатом перевірки реквізитів.

**1.5. Requester** – банк-відправник, який ініціює VoP перевірку.

**1.6. Responder** – банк-отримувач, який обробляє VoP запит та надає відповідь.

**1.7. Код ID НБУ (NBU ID)** – унікальний 6-значний ідентифікатор учасника СЕП НБУ.

**1.8. Match Status** – статус співпадіння реквізитів:
- **MATCH** – повне співпадіння
- **CLOSE_MATCH** – часткове співпадіння
- **NO_MATCH** – не співпадає
- **NOT_SUPPORTED** – перевірка не підтримується
- **ERROR** – технічна помилка

**1.9. SLA (Service Level Agreement)** – угода про рівень обслуговування, що визначає вимоги до доступності та продуктивності VoP.

---

## 2. ПРЕДМЕТ ДОГОВОРУ

**2.1.** НБУ надає Учаснику доступ до VoP системи для перевірки реквізитів отримувачів платежів перед відправленням платіжних повідомлень до СЕП НБУ.

**2.2.** Учасник зобов'язується:
- Інтегрувати VoP систему у свою інфраструктуру
- Забезпечити обробку VoP запитів від інших учасників (як Responder)
- Виконувати VoP перевірки перед відправленням платежів (як Requester)
- Дотримуватися вимог цього Договору та технічної специфікації VoP

**2.3.** Участь у VoP є:
- **Обов'язковою** для банків-учасників Миттєвих Переказів СЕП НБУ
- **Добровільною** для інших учасників СЕП НБУ (на період до 2028 року)

---

## 3. ПРАВА ТА ОБОВ'ЯЗКИ НБУ

**3.1. НБУ має право:**

3.1.1. Надавати Учаснику доступ до VoP Router для відправлення та отримання VoP запитів

3.1.2. Встановлювати та змінювати технічні вимоги до VoP інтеграції (з попередженням за 30 днів)

3.1.3. Проводити аудит VoP системи Учасника для забезпечення відповідності вимогам

3.1.4. Призупиняти доступ Учасника до VoP у випадку:
- Систематичних порушень SLA
- Загроз безпеці VoP системи
- Невиконання вимог цього Договору

3.1.5. Вимагати від Учасника усунення виявлених недоліків протягом встановленого терміну

3.1.6. Збирати та аналізувати статистику використання VoP для оптимізації системи

**3.2. НБУ зобов'язується:**

3.2.1. Забезпечити доступність VoP Router згідно з SLA (мінімум 99.9% uptime)

3.2.2. Надавати технічну підтримку Учаснику під час інтеграції та експлуатації VoP

3.2.3. Повідомляти Учасника про планові технічні роботи не пізніше ніж за 7 днів

3.2.4. Забезпечувати конфіденційність даних, що передаються через VoP Router

3.2.5. Надавати Учаснику:
- Документацію API
- Тестове середовище (sandbox)
- Сертифікати для mTLS автентифікації
- Облікові дані OAuth 2.0

---

## 4. ПРАВА ТА ОБОВ'ЯЗКИ УЧАСНИКА

**4.1. Учасник має право:**

4.1.1. Отримувати технічну підтримку від НБУ з питань VoP інтеграції

4.1.2. Запитувати роз'яснення щодо технічної специфікації VoP

4.1.3. Брати участь у робочих групах з розвитку VoP

4.1.4. Отримувати статистику власних VoP запитів та відповідей

**4.2. Учасник зобов'язується:**

**4.2.1. Як Requester (банк-відправник):**
- Виконувати VoP перевірку перед відправленням платежів згідно з правилами
- Відображати результат VoP перевірки клієнту
- Логувати всі VoP запити для аудиту (мінімум 90 днів)
- Не передавати третім сторонам дані, отримані від VoP

**4.2.2. Як Responder (банк-отримувач):**
- Забезпечити доступність VoP Responder API згідно з SLA (99.9% uptime)
- Обробляти VoP запити протягом максимум 1 секунди (p95)
- Повертати коректні результати співставлення імен
- Захищати дані клієнтів від несанкціонованого доступу
- Логувати всі VoP запити для аудиту (мінімум 90 днів)

**4.2.3. Безпека:**
- Використовувати mTLS для автентифікації
- Використовувати OAuth 2.0 + FAPI для авторизації
- Використовувати QWAC або АЦСК сертифікати
- Забезпечити безпечне зберігання приватних ключів (HSM/KMS)
- Негайно повідомляти НБУ про інциденти безпеки

**4.2.4. Compliance:**
- Дотримуватися вимог GDPR та Закону України "Про захист персональних даних"
- Не передавати персональні дані клієнтів третім сторонам без згоди
- Мінімізувати обсяг даних, що передаються через VoP
- Забезпечити право клієнтів на доступ, виправлення, видалення їхніх даних

**4.2.5. Моніторинг та звітність:**
- Моніторити доступність та продуктивність власних VoP систем
- Надавати НБУ щомісячну статистику VoP (за запитом)
- Повідомляти НБУ про технічні проблеми протягом 1 години після виявлення

## 5. ТЕХНІЧНІ ВИМОГИ

**5.1. Requester API вимоги:**

5.1.1. Інтеграція з VoP Router через mTLS + OAuth 2.0
5.1.2. Формування VoP Request згідно з JSON Schema
5.1.3. Обробка VoP Response та відображення результату клієнту
5.1.4. Timeout для VoP запиту: максимум 3 секунди

**5.2. Responder API вимоги:**

5.2.1. Endpoint: `POST /vop/v1/verify`
5.2.2. Протокол: HTTPS (TLS 1.3)
5.2.3. Автентифікація: mTLS (client certificate)
5.2.4. Авторизація: OAuth 2.0 Bearer token
5.2.5. Формат: JSON (RFC 8259)
5.2.6. Максимальний час відповіді: 1000 мс (p95), 3000 мс (p99)
5.2.7. Доступність: 99.9% uptime

**5.3. Name Matching алгоритми:**

5.3.1. Використання Levenshtein Distance або Jaro-Winkler Distance
5.3.2. Пороги співставлення:
- MATCH: similarity ≥ 95%
- CLOSE_MATCH: 75% ≤ similarity < 95%
- NO_MATCH: similarity < 75%

**5.4. Сертифікати:**

5.4.1. Тип: QWAC (Qualified Website Authentication Certificate) або АЦСК
5.4.2. Key Type: RSA 2048-bit або ECDSA P-256
5.4.3. Validity: максимум 2 роки
5.4.4. Revocation: підтримка CRL або OCSP

## 6. SERVICE LEVEL AGREEMENT (SLA)

**6.1. VoP Router (НБУ):**

| Метрика | Target | Measurement |
|---------|--------|-------------|
| Uptime | 99.9% | Щомісяця |
| Latency (p95) | < 500 мс | Real-time |
| Latency (p99) | < 1000 мс | Real-time |
| Max downtime | 43.8 хв/місяць | Щомісяця |

**6.2. VoP Responder (Учасник):**

| Метрика | Target | Measurement |
|---------|--------|-------------|
| Uptime | 99.9% | Щомісяця |
| Latency (p95) | < 1000 мс | Real-time |
| Latency (p99) | < 3000 мс | Real-time |
| Error rate | < 2% | Щомісяця |

**6.3. Штрафні санкції за порушення SLA:**

Якщо Учасник систематично (3 місяці поспіль) не виконує SLA:
- 1-й місяць: попередження
- 2-й місяць: письмове попередження з вимогою усунення
- 3-й місяць: призупинення доступу до VoP

---

## 7. ВАРТІСТЬ ПОСЛУГ

**7.1.** Участь у VoP системі є **безкоштовною** для всіх учасників СЕП НБУ.

**7.2.** Учасник самостійно несе витрати на:
- Розробку та інтеграцію VoP Requester/Responder APIs
- Отримання QWAC/АЦСК сертифікатів
- Інфраструктуру (сервери, мережа, тощо)
- Підтримку та експлуатацію VoP систем

---

## 8. КОНФІДЕНЦІЙНІСТЬ ТА ЗАХИСТ ДАНИХ

**8.1.** Сторони зобов'язуються забезпечити конфіденційність інформації, отриманої в рамках цього Договору.

**8.2.** Учасник зобов'язується:
- Обробляти персональні дані клієнтів виключно для цілей VoP перевірки
- Не зберігати персональні дані довше ніж 90 днів (тільки для аудиту)
- Не передавати дані третім сторонам без згоди НБУ та клієнта
- Використовувати шифрування для зберігання та передачі даних

**8.3.** НБУ зобов'язується:
- Не зберігати персональні дані клієнтів у VoP Router (pass-through only)
- Зберігати тільки метадані запитів (без імен та IBAN) для статистики
- Забезпечити захист VoP Router від несанкціонованого доступу

---

## 9. ВІДПОВІДАЛЬНІСТЬ СТОРІН

**9.1.** НБУ не несе відповідальності за:
- Збитки Учасника або його клієнтів внаслідок неправильної інтеграції VoP
- Неточність даних, що повертаються Responder
- Технічні проблеми на стороні Учасника

**9.2.** Учасник несе відповідальність за:
- Точність даних клієнтів у власній базі даних
- Коректність роботи VoP Responder API
- Збитки клієнтів внаслідок неправильної VoP перевірки

**9.3.** У випадку порушення умов Договору винна Сторона зобов'язана відшкодувати збитки іншій Стороні.

---

## 10. ТЕРМІН ДІЇ ТА ПРИПИНЕННЯ ДОГОВОРУ

**10.1.** Цей Договір набуває чинності з дати підписання та діє безстроково.

**10.2.** Договір може бути розірваний:

10.2.1. За взаємною згодою Сторін

10.2.2. В односторонньому порядку НБУ у випадку:
- Систематичних порушень Учасником умов Договору
- Загроз безпеці VoP системи з боку Учасника
- Втрати Учасником ліцензії НБУ на здійснення банківської діяльності

10.2.3. В односторонньому порядку Учасником у випадку:
- Припинення банківської діяльності
- З попередженням за 90 днів (тільки для добровільних учасників)

**10.3.** У випадку розірвання Договору Учасник зобов'язаний:
- Припинити використання VoP Router протягом 30 днів
- Видалити всі персональні дані, отримані через VoP
- Повернути сертифікати та облікові дані
